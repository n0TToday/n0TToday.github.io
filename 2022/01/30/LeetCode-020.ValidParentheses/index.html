2. 思路

记得当时学习数据结构的时候，括号问题就曾经出现在练习题还是教材书中，左括号与右括号配对应该是符合先进后出的特性的，队列和栈中应该使用栈实现是比较好的方法。

但是 C# 学习的程度不深，没有学习 栈(Stack) 的使用，于是第一遍自己做出了下面这个比较麻烦的方法：

2.1 利用KVP模拟栈

因为一开始不知道栈怎么用，而第一遍又想自己思考着解决，于是利用了前几天刚学的 键值对 Dictionary 模拟栈来操作，毕竟也有 ADD 之类的方法嘛~

  • 将字符串转为 char[] 类型。

    这一步好像不需要，直接遍历 string 也可以。

  • 判断数组长度是否为空 或 是否为奇数。

    • 如果是空的或者是奇数就结束，返回判断值。

  • 新建 Dictionary。

  • 遍历数组。

    • 如果是 左括号 ，添加进键值对。
    • 如果是 右括号 ，取出最近添加进 KVP 的值，比较是否为一对。
      • 是，在 KVP 中 Remove 掉这个右括号，继续遍历下一个。
      • 否，结束遍历，返回判断值。

  • 遍历完后，判断 KVP 是否为空。

    • 如果不为空，则有未配对的括号，返回 False 。

写完自己的这个方法后，忽然发现好像不需要用键值对，简单的一维数组也能解决 = =

2.2 栈解决

栈解决这个问题就方便了很多，毕竟有现成的方法。另外还在评论区发现了更便利的方法，减少了判断次数，属实让我这个小白大开眼界了：

  • 判断字符串长度是否为空 或 是否为奇数。

  • 创建一个类型为 char 的新栈。

  • 遍历字符串。

    • 如果为左括号，则添加 对应的右括号 进栈。

    • 如果是右括号，则 栈顶出栈 同时判断是否与出栈元素相等。

      这里真的有点妙，这种自己没想到的方法的惊喜和冲击让我更爱刷 LeetCode 了~

      • 另外如果这是栈内为空，则说明有不匹配括号。

  • 遍历完成后，返回 栈内是否为空的判断结果

    这里也很妙~ return stack.Count == 0;

2.3 字符串法

一个更另类和取巧的思路，可惜照着实现了一遍，性能有些差劲。

  • 照例判断 空和奇数。

  • while 判断字符串是否包含 [] () {} 。

    只要是成对的字符串，一定包含以上三个之一。

    • 如果有，则将其 replace 为空。

  • 返回字符串是否为空。

不得不说这个方法真的简单和聪明，不知道什么时候能自己想出来，啧。
